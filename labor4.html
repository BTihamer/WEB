Labor 2 - Saját "domain", HTML, CSS gyakorlás

Feladatok + házi

Órai cél

A diákok készítsenek egy személyes honlapot a saját felhasznalo.nev.github.io domain alatt az AcademicPages sablon alapján, és a _pages/lab1.html oldalon CSS segítségével alakítsanak ki egy sakktáblát.

Előfeltételek

  • Aktív GitHub-fiók. Azonosítót az osztálylistába kérjük. Ide írd be
  • Alap Git ismeret: klónozás, commit, push, branch.
  • Egy portré vagy avatar kép (ha nem szeretnék saját fotóval).
  • Alap mappa strukrutra létrehozása (ezt meg lesz a fork-ból)

1) Saját domain létrehozása

  1. Fork az AcademicPages-repo-ról: academicpages/academicpages.github.io.
  2. A fork átnevezése saját felhasználónévre: felhasznalo.github.io.
  3. GitHub Pages bekapcsolása a repo beállításainál (Settings → Pages). Forrás: Deploy from a branch, branch: main.
  4. Várni a buildre, majd teszt: https://felhasznalo.github.io. vagy név. pl. tihamerballa.github.com.

Ha org alatt dolgoznak, maradhat a repo felhasznalo.github.io néven publikusan.

2) Minimál beállítások

A) Alap adatok

  • _config.yml: title, email, description, url (pl. https://felhasznalo.github.io), baseurl: ""
  • Profilkép: másolják a képet az images/ mappába, majd állítsák be a kezdőlapon.
  • Elérhetőségek: email, LinkedIn, GitHub linkek kitöltése.

B) A következőket törölni kell:

A következőket kérjük eltüntetni: Publications, Talks, Teaching.

  • Navigáció: a legtöbb AcademicPages-változatnál a menü a _data/navigation.yml-ben van. Kommentezzék ki vagy töröljék a fenti tételeket.
  • Oldalak: a kapcsolódó fájlok általában a _pages/ mappában vannak. Hagyhatják meg, de tegyék nav_exclude: true-ra az oldalak front matter-ében, vagy ideiglenesen törölhetik őket.
# példa a _data/navigation.yml-ből

  url: /

3) Kezdőoldal minimális testreszabása

  • Fénykép vagy avatar megjelenítése a nyitóoldalon.
  • Név és rövid bemutatkozás (1–2 mondat).
  • Elérhetőségek: email + legalább egy közösségi/portfólió link.
  • 1–2 tartalmi blokk a hallgató érdeklődési köreiről.

Portfolio oldal: töltsenek fel 3–6 képet, amelyek az egyetemi élethez kapcsolódnak. Írjanak rövid (1–2 mondatos) képaláírást, és ügyeljenek az alt szövegre (akadálymentesség).

5) Feladat 3 – CSS-sel alakítsanak ki sakktáblát

Csak a <style> részben dolgozzanak; a HTML-t ne módosítsák. Értékelésnél díjazzuk a kreatív, de tiszta megoldásokat (változó színek, hover, árnyék, animáció, címkék az élek mentén stb.).

  • Alap: 8×8 tábla jól látható, kontrasztos mezőkkel.
  • Pluszpont: reszponzív méretezés (vw alapú méretek), fókuszállapotok, magas kontraszt mód.

Házifeladat

1.) (válassz legalább 1-et)

  1. Rólunk/Profil oldal készítése Markdownban (front matter-rel: title, permalink). Legyen rajta rövid bemutatkozás és egy kép alt szöveggel.
  2. Kapcsolat szekció a nyitóoldalon: mailto link + 2 közösségi profil ikon/link. (Akadálymentes címkék: aria-label.)
  3. Menü testreszabás: _data/navigation.yml szerkesztése – a felesleges elemek (Publications/Talks/Teaching) rejtve maradnak, a sorrend pedig saját igény szerint.

2.) (válassz legalább 1-et)

  1. 404 oldal létrehozása (egyedi üzenettel és vissza a főoldalra gombbal).
  2. Favicon & site icon beállítása (favicon.ico vagy PNG + <link rel="icon"> a sablonban).
  3. Képek optimalizálása: legfeljebb 150KB/kép, loading="lazy" használata a portfólió képein.
  4. Dark mode csak CSS-sel: @media (prefers-color-scheme: dark) – színek átállítása külön JS nélkül.
  5. SEO meta: oldalcímek konzisztens formázása + leírás (description a _config.yml-ben), és megosztási kép (OG image) hozzáadása.

3.)

  1. Egyszerű rácsos portfólió (CSS Grid): 6 kártya, képaláírással (<figure> / <figcaption>), mobilon 2 oszlop, asztalin 3.
  2. Hozzászólás a kódhoz: nyissatok egy saját pull requestet a repo-ban, írjatok értelmes PR-leírást, majd merge-eljétek.
  3. Lab1 gyorslink: a főoldalra kerüljön egy „Sakktábla – Labor 1” gomb, amely a /lab1/ oldalra mutat (nav_exclude: true maradhat).

Ihlet: academicpages.github.io – de a megvalósítás legyen saját és egyszerű.

Mai óra célja:

  • GitHub azonosítók begyűjtve az osztálylistához.
  • Fork → átnevezés → Pages engedélyezve → oldal elérhető.
  • Minimál testreszabás: fotó/avatar, név, elérhetőség(ek), rövid bemutatkozás.
  • Nem kívánt menüelemek rejtése megoldva.
  • _pages/lab1.html létrehozva, sakktábla CSS legalább alap szinten kész.
